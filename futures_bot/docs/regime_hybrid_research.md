# regime_hybrid 전략 실험 기록 (2026-07-08 ~ 2026-07-10)

`strategies/regime_hybrid.py`는 `trend_breakout`을 대체하기 위해 시도한 실험용 전략이다.
**`main.py`, `run_backtest.py`는 여전히 `trend_breakout`만 사용한다 — 이 문서의 실험은
실거래 코드에 아무 영향을 주지 않았다.** regime_hybrid는 아직 백테스트에서도 순손실
상태라 실거래에 연결하지 않았다.

## 왜 시작했나

`trend_breakout`(EMA추세+볼린저돌파+RSI모멘텀)을 108개 파라미터 조합으로 그리드서치했지만
BTC/USDT 15분봉 270일 TRAIN/VAL/TEST 전 구간에서 VALIDATION 기준 플러스가 하나도 없었다.
TEST 자본곡선이 급락이 아니라 완만한 우하향이라 "레인지장에서 가짜돌파에 계속 걸린다"는
가설을 세우고, ADX로 추세장/레인지장을 나눠 서로 다른 로직을 쓰는 `regime_hybrid`를
새로 만들었다(추세장=trend_breakout과 동일 로직, 레인지장=반대 방향 평균회귀).

## 검증 방법론

모든 실험은 같은 원칙을 지켰다:
1. TRAIN(60%) / VALIDATION(20%) / TEST(20%) 3분할, TEST는 최종 확인에만 1회 사용
2. 파라미터/로직을 하나 바꿀 때마다 나머지는 전부 고정한 상태로 A/B 비교 (여러 개를 한번에
   바꾸면 뭐가 효과 있었는지 알 수 없기 때문)
3. VALIDATION에서 좋아 보여도 TEST에서 재현 안 되면 과최적화로 간주하고 채택하지 않음

|---|---|---|---|
| 0 | trend_breakout, 15분봉, 파라미터 튜닝만 | -15.73% | 파라미터 문제 아님 확인 |
| 1 | ADX 레짐필터 추가 (regime_hybrid 최초 버전) | -14.05% | 미미한 개선 |
| 2 | 15분봉 → 1시간봉 (거래빈도 감소) | -3.73%(VAL은 -14%대로 불안정) | 마찰비용 비중은 줄었으나 TRAIN/VAL 불안정 |
| 2-1 | 마찰비용(수수료+슬리피지) 분해 분석 | 손실의 26~94%가 마찰 비용 (구간별 편차 큼) | "수수료가 전부는 아님" 확인 — 순수 신호도 약함 |

| 5 | 레인지장 로직을 **%B+II%**로 교체 | 3구간 전부 악화 | ❌ 기각, RSI 단독으로 원복 |
| 6 | **가격-RSI 다이버전스** 추가 (lookback=3) | TRAIN -47.91%, 거래 5배 폭증 | ❌❌ 심각한 악화 — 원복 |
| 6-1 | 다이버전스 lookback 재탐색(5~20) | 크게 하면 신호가 아예 안 뜸, 작게 하면 여전히 노이즈 | ❌ 쓸모있는 중간지점 없음 — OFF 유지 |

## 최종 확정 상태 (regime_hybrid, 미사용 중)

```
timeframe = 1h
adx_trend_threshold = 30
squeeze_enabled = True, squeeze_percentile = 0.2
mfi_enabled = True
mr_use_pb_ii = False (RSI 단독 유지)
divergence_enabled = False
```

BTC/USDT, 1000일, 이 설정 기준:

| | 거래횟수 | 승률 | Profit Factor | MDD | 총수익률 |
|---|---|---|---|---|---|
| TRAIN | 73 | 27.4% | 0.89 | 17.5% | -4.26% |
| VALIDATION | 39 | 33.3% | 1.10 | 6.7% | **+0.69%** |
| **TEST** | 44 | 25.0% | 0.82 | 11.7% | **-4.04%** |

**여전히 TEST 기준 순손실이라 실거래에 채택하지 않는다.** 다만 최초 trend_breakout
15분봉 대비(-15.73%) 손실 폭을 크게 줄였고, 유효한 개선 1건(스퀴즈)을 찾은 것 자체가
"AND로 묶은 지연지표만으로는 안정적 우위를 만들기 어렵다"는 가설을 재확인해줬다.
