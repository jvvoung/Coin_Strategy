# 65개 전략 코드 분석표 (strategy_analysis_table.md)

### 재사용하면 위험한 전략 유형
1. **무한매수/매직스플릿류 (8, 9, 10, 12, 13, 18, 55, 62)**: 회차 기반 물타기 구조 자체가 "레버리지 없는 현물 DCA"를 전제로 함. 선물 레버리지 환경에 그대로 옮기면 물탄 만큼 명목 포지션이 커져 청산가가 빠르게 근접하는 구조적 위험이 있음 — 반드시 최대 회차/최대 명목가 상한과 하드 손절을 별도로 얹어야 함.
2. **저빈도 자산배분류 (1~7, 14, 22, 34, 35, 38~40, 42, 44, 45, 47, 49 등 약 20개)**: 월 1회 리밸런싱 + 손절 없음 전제. 선물 24시간 변동성·펀딩비 환경에서는 리밸런싱 주기 사이의 급락에 무방비로 노출됨.
3. **한국 특유 제도 기반 전략 (56번 SPAC)**: 한국 스팩 하방보장 제도에 전적으로 의존하는 구조라 코인 선물에 개념적으로 이식 불가.
4. **명시적 손절가가 없는 전략 전반**: 65개 중 다수가 "추세 이탈시 매도"만 있고 가격 기반 스탑로스가 없음. 이 패턴을 선물에 그대로 가져오면 급변동 구간에서 손절이 늦어져 청산으로 직행할 위험이 큼.

### 최종 통합 전략에 반영할 만한 공통 패턴
1. **이동평균 정배열/골든·데드크로스**가 사실상 전 폴더의 최소공통분모 추세 필터로 반복 사용됨 → `indicators/moving_average.py` + `strategies/trend_following.py`의 기본 골격으로 채택.
2. **RSI는 역추세(평균회귀)가 아니라 추세 방향 확인·모멘텀 필터로 일관되게 사용됨** → 통합 전략에서도 RSI를 진입 필터로 쓰되, 별도의 순수 평균회귀 모듈은 새로 설계해야 함(기존 코드에 참고할 원형이 없음).
3. **거래대금/거래량 기반 유니버스 필터링**(15, 31, 51, 52, 58, 61 등)이 다수 종목 운용 시 반복 등장 → 다중 심볼 운용 시 유동성 필터로 표준화.
4. **회차 기반 분할진입(최대 회차 제한)+목표수익 전량청산** 구조(8~13, 18, 28, 29)가 반복 → 포지션 사이징 모듈의 "단계적 진입" 옵션으로 표준화하되, 회차 상한과 하드 손절을 반드시 결합.
5. **카나리아자산/BTC 추세 필터로 전체 익스포저를 온오프하는 레짐필터**(6, 7, 15, 47, 49, 51, 52, 58)가 자산배분·알트코인 전략 모두에서 반복 → 통합 전략의 "시장 레짐 필터" 레이어로 채택 가능.
6. **트레일링스탑+분할익절 조합**(8, 9, 10, 12, 46)이 리스크관리 방식 중 상대적으로 정교한 축 → `risk/take_profit.py` 설계에 직접 반영.
7. **연속손절 발생시 포지션 축소(시스템손절)**(37번) → `risk/account_guard.py`의 연속 손실 제한 로직 설계에 참고.
8. **바이낸스 선물 Hedge Mode(positionSide LONG/SHORT) 실전 구현 사례**(10, 13, 33, 43, 48)가 이미 다수 확보되어 있어, 이 프로젝트의 `execution/` 및 `risk/leverage_manager.py` 설계의 실질적 출발점으로 삼을 수 있음.
9. **명시적 가격 기반 손절(스탑로스)의 전반적 부재**가 65개 전체를 관통하는 가장 큰 공통 약점 → 통합 전략에서는 모든 하위 전략에 공통으로 적용되는 손절 레이어(`risk/stop_loss.py`)를 전략 로직과 무관하게 강제해야 함.
